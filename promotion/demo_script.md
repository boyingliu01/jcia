### 第三部分：测试选择演示（3分钟）

**【演讲者】**

> 了解了影响范围后，下一个关键问题是：**需要测试哪些用例？**
>
> 传统做法是运行全部回归测试，耗时4-8小时。
> JCIA 提供了智能测试选择功能，可以基于影响分析结果，只选择相关的测试用例。

**【命令演示】**

```bash

jcia test --repo-path . --target-class com.example.Service --strategy impact_based --output test_selection.json
```

**【演讲者】**

> 我们运行了测试选择命令，JCIA 基于刚才的影响分析结果，智能选择了相关的测试用例。

**【展示测试结果】**

> 可以看到测试选择结果：
> - **全量测试用例**：500个
> - **JCIA智能选择**：120个
> - **选择比例**：24%
>
> 这意味着我们只需要运行24%的测试用例，就可以覆盖主要风险点！
> 按每个测试用例平均30秒计算：
> - 全量测试：500 × 30秒 = 4.1小时
> - JCIA选择：120 × 30秒 = 1小时
>
> **节省时间：3.1小时（75%）**

### 第四部分：CI/CD集成展示（3分钟）

**【演讲者】**

> JCIA 不仅可以本地使用，还可以无缝集成到 CI/CD 流程中。
> 这样每次代码提交时，都可以自动进行影响分析和测试选择。

**【展示Jenkins集成】**

```groovy
// Jenkins Pipeline 示例
pipeline {
    agent any
    stages {
        stage('Impact Analysis') {
            steps {
                sh '''
                    jcia analyze \
                        --repo-path . \
                        --from-commit ${env.GIT_PREVIOUS_COMMIT} \
                        --to-commit ${env.GIT_COMMIT} \
                        --output-format html \
                        --output-dir reports/
                '''
            }
        }
        stage('Test Selection') {
            steps {
                sh '''
                    jcia test \
                        --repo-path . \
                        --strategy impact_based \
                        --output test-selection.json
                '''
            }
        }
        stage('Run Tests') {
            steps {
                sh 'mvn test -Dtest=$(cat test-selection.json | jq -r ".selected_tests | join(\",\")")'
            }
        }
    }
    post {
        always {
            publishHTML([
                allowMissing: false,
                alwaysLinkToLastBuild: true,
                keepAll: true,
                reportDir: 'reports',
                reportFiles: 'impact_analysis.html',
                reportName: 'JCIA Impact Analysis Report'
            ])
        }
    }
}
```

**【演讲者】**

> 这是一个典型的Jenkins Pipeline配置，展示了如何在CI/CD流程中集成JCIA：
>
> **Stage 1 - 影响分析**：
> - 在每次代码提交后自动运行
> - 分析本次提交的影响范围
> - 生成HTML报告
>
> **Stage 2 - 测试选择**：
> - 基于影响分析结果
> - 智能选择相关测试用例
>
> **Stage 3 - 执行测试**：
> - 只运行选中的测试用例
> - 大幅缩短测试时间
>
> **Post - 报告发布**：
> - 将JCIA报告集成到Jenkins界面
> - 方便团队查看

## 附录：常见问题准备

### Q1: JCIA支持哪些编程语言？
**A:** 目前主要支持Java，计划后续支持Kotlin和Scala等JVM语言。

### Q2: 分析大型项目会不会很慢？
**A:** JCIA针对大型项目进行了性能优化。以100万行代码的项目为例，完整分析通常在5-10分钟内完成。我们还支持增量分析模式，只分析变更部分，速度更快。

### Q3: 准确率如何？会不会漏掉影响点？
**A:** JCIA采用静态分析和动态分析相结合的方式，通过多策略融合引擎，准确率达到85%以上。对于关键路径，我们会通过多种策略交叉验证，确保重要影响点不被遗漏。

### Q4: 能否集成到我们现有的Jenkins流程？
**A:** 完全可以。JCIA提供了Jenkins Pipeline插件和CLI工具，可以轻松集成到现有的CI/CD流程中。我们提供了详细的集成文档和示例配置。

### Q5: 是否支持私有部署？
**A:** 是的，JCIA支持完全私有部署，所有代码分析都在本地进行，不会将代码上传到任何外部服务器，确保代码安全。

### Q6: 学习成本高吗？
**A:** JCIA的设计目标就是简单易用。基本的分析只需要一条命令即可运行。对于高级功能，我们提供了详细的文档和视频教程，通常30分钟即可上手。

### Q7: 是否收费？
**A:** JCIA是开源项目，采用MIT许可证，完全免费使用。
