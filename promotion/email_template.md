## 它能解决什么问题？

在日常开发和版本发布过程中，我们经常遇到这些痛点：

- 修改了一行代码，不确定会影响哪些功能模块
- 回归测试时需要运行全部用例，耗时4-8小时
- 代码审查时难以快速评估变更影响范围
- 担心遗漏某些受影响的功能点，导致线上问题

**JCIA 正是为解决这些问题而生。**

## 核心能力一览

| 能力 | 说明 | 效果 |
|:---:|------|------|
| 自动影响分析 | 识别代码变更的影响范围 | 节省90%分析时间 |
| 智能测试选择 | 基于影响范围选择相关测试用例 | 减少70%测试用例 |
| 多维度评估 | 6维度严重程度评分 | 精确定位风险 |
| CI/CD集成 | 支持Jenkins、GitLab CI | 无缝融入现有流程 |

## 典型使用场景

**场景1：日常开发 - 提交前自检**
```
开发者提交代码前，运行 JCIA 分析影响范围
确保不会引入意外影响
```

**场景2：代码审查 - 辅助评审**
```
代码审查时，自动展示变更影响报告
帮助评审者快速理解变更范围
```

**场景3：回归测试 - 测试选择**
```
版本发布前，智能选择相关测试用例
避免全量测试，缩短发布时间
```

### 邮件正文

---

**各位 JCIA 用户，大家好！**

我们很高兴宣布 **JCIA v2.0** 正式发布！本次更新带来了多项重磅功能，进一步提升了影响分析的准确性和实用性。

## 本次更新亮点

### 1. 混合分析融合引擎

**什么是混合分析？**

JCIA现在支持多种分析策略的融合：
- 贝叶斯融合策略
- 投票融合策略
- 加权融合策略

**优势：** 通过多策略融合，分析准确率提升35%，误报率降低50%。

### 2. 多维度严重程度评估

**6维度评分系统：**

| 维度 | 说明 | 权重 |
|------|------|------|
| 调用深度 | 变更方法在调用链中的深度 | 20% |
| 影响范围 | 受影响的类/方法数量 | 25% |
| 业务关键度 | 核心业务/非核心业务的区别 | 25% |
| 数据敏感性 | 涉及敏感数据操作的程度 | 15% |
| 历史缺陷 | 该区域的缺陷历史 | 15% |

**输出结果：** 每个变更都会获得一个综合严重度评分（1-10分）

### 3. 深度调用链追踪增强

- 最大追踪深度从5层增加到10层
- 支持递归调用检测
- 新增循环调用标记

### 4. CI/CD集成优化

- 新增Jenkins Pipeline插件
- 支持GitLab CI/CD原生集成
- 提供SonarQube插件

## 升级指南

### 从v1.x升级到v2.0

```bash
# 方式1：pip升级
pip install --upgrade jcia

# 方式2：源码升级
git pull origin main
pip install -r requirements.txt
```

### 配置文件更新

v2.0引入了新的配置项，建议更新`.jcia.yaml`：

```yaml
# 新增：混合分析配置
analysis:
  fusion_strategy: "weighted"  # bayesian/voting/weighted
  dimensions:
    - depth
    - scope
    - criticality
    - sensitivity
    - history

# 新增：严重程度评估
severity:
  weights:
    depth: 0.20
    scope: 0.25
    criticality: 0.25
    sensitivity: 0.15
    history: 0.15
```
