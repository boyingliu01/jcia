### 【海报左上】核心价值

#### 一句话描述
**自动化识别代码变更影响范围，智能选择测试用例，让回归测试更高效**

#### 3大核心价值

| 图标 | 价值 | 说明 |
|:---:|------|------|
| ⚡ | **节省时间** | 自动分析替代人工梳理，减少80%分析时间 |
| 🎯 | **精准测试** | 智能选择相关测试用例，避免全量测试 |
| 🔒 | **降低风险** | 全面识别影响范围，减少遗漏风险 |

### 【海报左下】使用场景

#### 场景1：日常开发 - 代码提交前自检
```
开发者提交代码前，快速分析影响范围
确保不会引入意外影响
```

#### 场景2：代码审查 - 辅助评审决策
```
代码审查时，自动展示变更影响
帮助评审者快速理解变更范围
```

#### 场景3：回归测试 - 测试用例选择
```
版本发布前，智能选择相关测试用例
避免全量测试，缩短发布时间
```

#### 场景4：故障排查 - 影响范围定位
```
生产环境出现问题，快速定位影响范围
辅助制定修复策略
```

### 【海报右下】效果对比

#### 使用前 vs 使用后

| 指标 | 使用前 | 使用后 | 改善 |
|------|--------|--------|------|
| 影响分析时间 | 2-4小时 | 5-10分钟 | **节省90%** |
| 测试用例数量 | 全量(100%) | 智能选择(20-30%) | **减少70%** |
| 回归测试时间 | 4-8小时 | 1-2小时 | **节省75%** |
| 遗漏风险 | 高(人工梳理) | 低(自动分析) | **显著降低** |

- 辅助色：橙色 (#f39c12) - 代表活力、效率
